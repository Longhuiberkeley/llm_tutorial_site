📋 What a SPEC Kit entry looks like

As a beginner chess player
I need to see a new puzzle each session and get a hint when I'm stuck
So that I can improve my tactical skills at my own pace
INPUT: User's skill level (set during onboarding), puzzle database
OUTPUT: One puzzle per session, styled board view, optional hint revealed on request
CONSTRAINTS: Puzzles must match skill level; hints shown max once per puzzle; no spoilers visible by default
SUCCESS: 70% of users solve the puzzle or engage with the hint in the first session

You cannot fill this out without knowing your user, input, output, rules, and success metric — exactly what vibe coding skips.
