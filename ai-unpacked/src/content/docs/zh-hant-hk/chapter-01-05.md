## 🎯 核心目標
- 貫徹始終：距離是 LLM 決定下一個詞語的依據。
- 觀察 LLM 如何尋找與上下文「接近」的詞語。
- 理解整個過程：自動完成 (Autocomplete) → 距離 (Distance) → 生成 (Generation)。

<div class="not-prose callout callout-tldr">

LLM 通過尋找與上下文「接近」的詞語來生成文本。「這美味的 ___」→ 披薩很接近，貓則很遠 → LLM 選擇披薩。

</div>

## 📝 關鍵概念

- **上下文 (Context)：** 提示詞 (Prompt) 在意義空間 (Meaning-space) 中建立初始坐標。
- **距離計算：** LLM 測量到其詞彙表中每個詞語的距離。
- **輸出最接近的詞語：** 它會統計性地選擇最接近且有效的詞語作為「下一個」詞語。
- **循環：** 這回到了超強自動完成 (Autocomplete) 的概念 —— 這正是生成過程的運作方式。

<div class="not-prose callout callout-dyk">

這就是為什麼 LLM 在寫作時無法「回頭修改」的原因 —— 它們只能向前看，根據之前出現的所有內容來預測下一個詞元 (Token) 。

</div>

<div class="not-prose callout callout-error">

LLM 不是基於理解來「選擇」詞語。它們是計算距離並進行統計選擇。沒有思考，只有數學。

</div>
