## 了解詞元 (Token) 定價

每個供應商都按**每百萬詞元 (Per Million Tokens)** 收費。一個詞元大約是 ¾ 個英文字詞 — 所以 100 萬個詞元 ≈ 750,000 個單詞。

定價有兩個獨立的費率：
- **輸入詞元 (Input tokens)** — 你發送給模型的內容（你的提示 (Prompt)、文件、上下文 (Context)）
- **輸出詞元 (Output tokens)** — 模型生成的內容（通常比輸入貴 3–4 倍）

在開發產品時，還有兩個重要的細節：

**推理詞元 (Reasoning tokens)** — 某些模型在產生答案前會消耗詞元來內部處理問題。這些計入輸出詞元並增加你的帳單。

**快取命中 (Cache hits)** — 如果你重複發送相同的大量上下文（例如系統提示或文件），供應商會將其快取。快取命中通常僅花費**標準輸入價格的 ~10%** — 這在大規模使用時能節省大量成本。

:::callout-dyk
訂閱方案由供應商**大幅補貼**以推動採用。在低到中等使用量下，通過 API 計費的相同使用量可能貴得多。如果你是個人用戶，這很棒 — 但在開發產品時要了解其中的區別。
:::

## 如何解讀 LLM 基準測試 (Benchmarks)

基準測試 (Benchmarks) 是對模型在特定任務上的表現進行評分的標準化測試。你會在評論和公告中隨處看到它們被引用。你可能會遇到的一些測試：

- **MMLU** — 測試 57 個學科（法律、醫學、歷史、數學……）的通識知識
- **HumanEval / MBPP** — 衡量編碼能力：模型能寫出正確的程式嗎？
- **GPQA** — 研究生級別的科學問題，旨在讓搜尋 Google 也難以回答
- **MATH** — 競賽級別的數學解題
- **Chatbot Arena (LMSYS)** — 真人比較兩個模型的回答並投票；ELO 分數反映了大比例真人的實際偏好

:::callout-error
**基準測試 (Benchmarks) 只是起點，而非定論。** 模型越來越多地針對基準測試相關數據進行訓練，從而虛報分數。更重要的是，一個在 MMLU 排行榜名列前茅的模型可能仍會在你的特定任務中出錯。最好的評估始終是你自己的評估：從你的使用場景中選取 10–20 個真實範例，並在不同模型中運行它們。你觀察到的結果勝過任何發布的數字。
:::
