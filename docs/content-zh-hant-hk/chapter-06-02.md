## 🎯 核心目標
- 了解將 LLM 連接到工具的行業標準。
- 明白 MCP 不僅僅是「獲取數據」——它還可以提供工具、數據和上下文 (Context)。

:::callout-tldr
在 MCP 出現之前，每個 AI 工具都需要編寫自定義代碼才能與每個 LLM 通訊。**MCP** 將此過程標準化，讓任何 AI 模型都能透過一個通用接口連接到任何工具、數據源或上下文提供者。
:::

## 📝 關鍵概念

- **標準化 (Standardization)：** 正如 USB-C 取代了數十種不同的充電線，MCP 取代了數百種自定義整合。任何 AI 模型、任何伺服器 —— 一個標準。
- **MCP 伺服器 (MCP Servers)：** 插入 AI 的小型程式。它們可以提供三種截然不同的功能：
  - **工具 (Tools)** —— AI 可以觸發的操作（例如：運行網頁搜尋、執行代碼、發送電子郵件）
  - **數據 (Data)** —— AI 可以讀取的文件、數據庫、GitHub 儲存庫
  - **上下文 (Context)** —— 預先加載的知識，如最佳實踐指南、旅遊規劃模板或領域術語表
- **MCP 客戶端 (MCP Clients)：** 「插入」這些伺服器的 AI 模型（如 Claude）。
- **AI 原生介面 (AI-native interfaces)：** 許多軟體工具現在都內建了 MCP 支援 —— 命令行工具、項目管理工具和 CRM 越來越多地提供 AI 就緒的介面，以便智能代理 (Agents) 可以直接操作它們。

:::callout-dyk
MCP 不僅僅是為了提取數據。公司可以創建一個 MCP 伺服器，加載其內部報銷政策、編碼標準或客戶服務手冊 —— 讓 AI 瞬間獲得上下文，而無需你將其粘貼到每個提示詞 (Prompt) 中。
:::

:::callout-tip
MCP 生態系統正在快速發展。值得搜尋「2025 年最佳 MCP 伺服器」或「Claude 使用什麼工具」來查看當前可用的工具 —— 從網頁瀏覽器和試算表編輯器到代碼運行器和日曆整合。你發現的東西可能會讓你感到驚訝。
:::
